	25. MVC 프레임워크 만들기 - 프론트 컨트롤러 패턴 소개
		* 프론트 컨트롤러 도입 전
			클라이언트1 호출 -> 컨트롤러 1
			클라이언트2 호출 -> 컨트롤러 2
			클라이언트3 호출 -> 컨트롤러 3
			
		* 프론트 컨트롤러 도입 전
			클라이언트1 호출 -> 공통 프론트 컨트롤러 -> 컨트롤러 1
			클라이언트2 호출 -> 공통 프론트 컨트롤러 -> 컨트롤러 2
			클라이언트3 호출 -> 공통 프론트 컨트롤러 -> 컨트롤러 3
			
			- 프론트 컨트롤러 서블릿 하나로 클라이언트의 요청을 받음
			- 프론트 컨트롤러가 요청에 맞는 컨트롤러를 찾아서 호출
			- 입구를 하나로!
			- 공통 처리 가능
			- 프론트 컨트롤러를 제외한 나머지 컨트롤러는 서블릿을 사용하지 않아도 됨
			
			- DispatcherServlet이 FrontColler 패턴으로 구현되어 있음.
